# 需求澄清 Skill

## 概述

本 Skill 将用户自然语言问题映射到**魔数���指标**上，基于映射结果改写用户问题，为后续阶段的向量匹配和关键词检索提供精准的关键词。

**核心价值**：
1. **问题规范化**：将用户问题改写为基于标准指标名称的描述
2. **指标映射**：将用户问题中的所有词组（包括机构、时间等）都映射到魔数师指标
3. **人机协作**：始终展示映射结果让用户确认或调整
4. **关键词提取**：输出精准的关键词列表用于后续检索

---

## 输入参数

**用户问题**（必填）：自然语言表述的数据分析需求

```
用户问题: "我想分析南京分行的小微企业贷款风险"
```

---

## 核心概念：统一映射到指标

**所有用户词组都映射到魔数师指标**，没有"维度"的概念。

| 用户词组 | 映射到的指标 | 说明 |
|----------|--------------|------|
| "贷款余额" | 借据余额 | 分析度量指标 |
| "风险" | 五级分类 | 分析度量指标 |
| "南京分行" | **二级账务机构名称** | 机构筛选指标 |
| "泰州分行" | **二级账务机构名称** | 机构筛选指标 |
| "2024年" | **统计日期** | 时间筛选指标 |

**关键理解**：
- "南京分行"、"泰州分行"、"扬州分行"等具体分行名称，都映射到同一个指标："二级账务机构名称"
- 机构、时间等筛选条件在魔数师中也是**指标**，不是单独的"维度"
- 所有映射结果都是魔数师指标，只是用途不同（分析度量 vs 筛选条件）

**目标**：对每个关键词找到对应的魔数师指标。

```
┌─────────────────────────────────────────────────────────────┐

│     → 获取表关联的魔数师指标列表                               │
│                                                              │
│  3. 汇总所有指标，去重 → matched_indicators                   │

│  示例：                                                       │
│  "风险" → 术语"五级分类" → 表"E_LN_LOAN_SUMMARY"              │
│        → 指标"五级分类"、"借据余额"、"逾期天数"...            │
│                                                              │
│  "南京分行" → 术语"二级账务机构名称" → 表"..."                │
│            → 指标"二级账务机构名称"                            │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  [2.2] 指标分类（用于展示）                                   │
├─────────────────────────────────────────────────────────────┤
│  根据指标用途进行分类（方便用户理解，但本质上都是指标）：        │
│                                                              │

│     - 借据余额、五级分类、逾期天数、贷款本金...                │
```

### 阶段 3：排序与筛选

**目标**：对匹配的指标进行评分排序，准备用户确认的选项列表。

```
┌─────────────────────────────────────────────────────────────┐
│  [3.1] 指标评分                                              │
├─────────────────────────────────────────────────────────────┤
│  对每个匹配的指标计算评分：                                    │
│                                                              │
│  评分维度（总分 1.0）：                                        │
│  - 别名关键词覆盖度 (0-0.4)：指标别名包含多少用户关键词        │
│  - 描述语义匹配度 (0-0.3)：指标描述与用户问题的语义相关性      │
│  - 主题路径相关性 (0-0.2)：指标所属主题路径与关键词的匹配度    │
│  - 字段映射可用性 (0-0.1)：指标是否有有效的字段映射            │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  [3.2] 筛选输出                                              │
├─────────────────────────────────────────────────────────────┤
│  按指标用途分组输出：                                          │
│                                                              │
│  分析指标（按评分排序）：                                      │
│  - Top 10-15 个分析度量指标                                   │
│  - 前 3-5 个默认勾选                                          │
│                                                              │
│  筛选指标（识别到的筛选条件）：                                 │
│  - 所有识别到的筛选指标都输出                                  │
│  - 默认全部勾选                                               │
│  - 记录用户的原始输入值（如"南京分行"）                        │
└─────────────────────────────────────────────────────────────┘
```

| 用户输入示例 | 映射到的指标 | 说明 |

## 适用场景

- 用户输入自然语言问题，需要映射到魔数师指标
- 用户问题包含机构、时间等筛选条件词组
- 需要将用户问题规范化为标准表述
- 为后续主题推荐和模板检索提供精准关键词

---

## 输出用途

本 Skill 的输出将用于：

1. **主题聚合**：从 `confirmed_indicators` 中提取 `theme_id`，聚合推荐主题
2. **模板检索**：使用 `extracted_keywords` 进行模板关键词匹配
3. **筛选条件生成**：使用 `filter_indicators` 生成模板的筛选条件
4. **问题规范化**：使用 `normalized_question` 作为推荐结果的描述
